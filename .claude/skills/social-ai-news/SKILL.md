## 执行流程

### 1. 抓取数据
```bash
python scripts/social_ai_news.py "{主题}"
```

### 2. 分析热点
- 统计关键词频率
- 识别最热门话题
- 提取关键数据

### 3. 生成文章
直接生成最终文章，保存到 `ai-reports/{YYYYMMDD}-{事件关键词}.md`

## 文章规范

### 标题要求

**事件 + 数字 + 钩子**，可选风格：

| 风格 | 示例 |
|------|------|
| 数字冲击 | `机器人马拉松50分钟夺冠，人类纪录被打破？` |
| 悬念式 | `具身智能融资4.55亿，机器人赛道彻底炸了` |
| 故事式 | `300台机器人跑马拉松，一半趴窝了` |
| 对比式 | `机器人跑得比人快，但跑完的不到一半` |
| 现场式 | `今天北京机器人马拉松开跑，差距有点大` |

根据内容选择最适合的风格，不要每次都用数字开头。

### 写作风格（随机切换）

每次生成时，随机选择一种风格：

**风格A - 热闹型**：
- 开头：先抛事件，再抖数字，制造惊讶感
- 语气：炸了、沸腾、杀疯了、卷起来了
- 结尾：引导读者思考或行动

**风格B - 故事型**：
- 开头：设置场景，像讲新闻故事
- 语气：娓娓道来，像跟朋友聊天
- 结尾：留下悬念或情感共鸣

**风格C - 分析型**：
- 开头：提出问题或现象
- 语气：冷静分析，但用大白话
- 结尾：给出判断或建议

**风格D - 现场型**：
- 开头：时间+地点+事件，像现场报道
- 语气：带现场感，有画面描述
- 结尾：点出意义或后续

### 格式要求

- **首行缩进**：`　　`（两个全角空格）
- **段落合并**：同类内容合并，不要碎片化
- **关键粗体**：数字、结论用 `**粗体**`
- **口语化**：避免术语堆砌
- **禁止**：emoji、小标题、固定模板

### 话题信息

根据文章内容，在正文末尾添加相关话题（参考来源之前）：

```
---
**话题：** #机器人马拉松 #具身智能 #AI新突破

**参考来源：**
1. [文章标题] - [链接]
```

话题选择原则：
- 从内容关键词提取
- 最多2-4个话题
- 话题要有热度、有人搜

### 人称视角

**用"我们"代替"你"**，站在人类视角来写：

| ❌ 不要写 | ✅ 要写 |
|----------|--------|
| 你不用担心机器人抢你的跑步金牌 | 我们不用担心机器人抢走跑步金牌 |
| 对普通人有什么影响？ | 对我们有什么影响？ |
| 你可能会觉得... | 我们可能会觉得... |
| 以后你可以用机器人帮你干活 | 以后我们可以用机器人帮我们干活 |

写作时要代入"我们人类"的身份，让读者有代入感。

### 参考来源格式

```
**参考来源：**
1. [文章标题] - [链接]
2. [文章标题] - [链接]
```

**风格A示例：**

```markdown
# 机器人跑得比人快？今天这场马拉松差距有点大

　　今天北京有个马拉松，参赛的不是人，是机器人。**300多台人形机器人**跑21公里，结果冠军跑出了**50分26秒**，直接比人类纪录快8分钟。但有意思的是，能跑完的不到一半，有的机器人刚起步就趴窝了，场面一度很搞笑。

　　这场2026北京人形机器人半程马拉松是全球首场人机同跑的比赛。冠军"闪电"跑得飞快，步频每秒3步以上，但稳定性不行——300台机器人参赛，跑完的不到一半。有的跑不到1公里就倒地，有的走两步开始"抽筋"，还有的电池耗尽直接停摆。**机器人跑得快但跑不完，我们人类跑得慢但能跑到底**。

　　专家说得好：有限性就是人类的护城河。机器人可以快，但我们能持续、能适应、能应对突发情况。这次比赛更像技术展示，证明机器人能在复杂环境长距离运动。**中国正把人形机器人当未来产业重点发展**，这场马拉松就是公开测试。对我们来说，短期内不用担心机器人抢走什么，长期看可能会多一个帮手。

---
**话题：** #机器人马拉松 #人形机器人 #AI突破

**参考来源：**
1. 机器人半马50分26秒夺冠 - https://xxx

*日期：2026-04-19*
```

## 关键原则

1. **标题多变**：不固定数字开头，根据内容选风格
2. **风格切换**：每次随机选择写作风格
3. **话题标签**：结尾添加2-4个相关话题
4. **首行缩进**：两个全角空格
5. **段落合并**：不要太碎
6. **关键粗体**：突出数字
7. **直接可用**：复制即可发布
